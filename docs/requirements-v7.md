# AI Daily Briefing v7 升级需求文档

## 背景

AI Daily Briefing v6 已稳定运行，芯片/半导体和机器人/具身智能两个类目已上线。但存在以下问题：
- 芯片/机器人采集信息量不足（各 3 条查询）
- 跨源聚类算法准确性有待提高
- 跨源关联缺乏深度分析（因果链、趋势追踪）

## 目标

### 功能目标
1. 扩大芯片/半导体采集信息（增加查询数量 + 增加数据源）
2. 扩大机器人/具身智能采集信息（增加查询数量 + 增加数据源）
3. 优化跨源聚类算法（提高聚类准确性）
4. 增加跨源关联深度分析（因果链、趋势追踪）

### 非目标
- 不新增专题报告章节
- 不更换数据源（仍以 Exa API 为主力）
- 不更换推送渠道（仍使用飞书）

## 验收标准

| 编号 | 验收标准 | 优先级 |
|------|----------|--------|
| AC-1 | 芯片/半导体查询从 3 条增加到 6-8 条 | P0 |
| AC-2 | 机器人/具身智能查询从 3 条增加到 6-8 条 | P0 |
| AC-3 | 新增芯片/机器人专项数据源（如 TechCrunch、IEEE Spectrum 等） | P0 |
| AC-4 | 跨源聚类算法优化（Jaccard 阈值调整、关键词权重） | P1 |
| AC-5 | 跨源关联增加因果链分析 | P1 |
| AC-6 | 跨源关联增加趋势追踪（3 天窗口） | P1 |
| AC-7 | 测试覆盖率 > 60% | P1 |
| AC-8 | 向后兼容（JSON 输出结构不变） | P0 |

## 风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 查询增加导致 API 配额超限 | 采集失败 | 监控 API 使用量，设置告警 |
| 聚类算法优化引入误判 | 聚类不准确 | 充分测试，保留回滚机制 |
| 因果链分析复杂度过高 | 性能下降 | 限制分析深度，异步处理 |
