# AI Daily Briefing v6 升级需求文档

## 背景

当前 AI Daily Briefing v5 已稳定运行，但存在以下问题：
- 分类覆盖不足：缺少"芯片/半导体"和"机器人/具身智能"两个热门领域的专项分类
- 代码架构：单文件 660 行，难以维护和扩展
- 代码质量：缺少类型注解、错误处理不够完善
- 配置管理：查询参数硬编码，难以调整

## 目标

### 功能目标
1. 新增"芯片/半导体"分类 + 专项采集查询
2. 新增"机器人/具身智能"分类 + 专项采集查询
3. 报告模板增加对应章节

### 质量目标
4. 代码质量：类型注解完整、ruff 零错误、错误处理完善
5. 架构优化：模块化拆分、配置外置、日志系统
6. 功能增强：去重算法优化、报告模板升级

### 约束条件
7. 向后兼容：现有 cron job 和飞书推送不受影响
8. 输出格式：JSON 结构保持兼容（可新增字段，不删除现有字段）

## 验收标准

| 编号 | 验收标准 | 优先级 |
|------|----------|--------|
| AC-1 | 新增"芯片/半导体"分类，自动归类相关新闻 | P0 |
| AC-2 | 新增"机器人/具身智能"分类，自动归类相关新闻 | P0 |
| AC-3 | 新增 6+ 条专项采集查询（芯片 3 条 + 机器人 3 条） | P0 |
| AC-4 | 报告模板包含芯片和机器人专题章节 | P0 |
| AC-5 | ruff 错误数 = 0 | P0 |
| AC-6 | 类型注解覆盖率 > 80% | P1 |
| AC-7 | 单元测试覆盖率 > 60% | P1 |
| AC-8 | 配置文件外置（YAML/JSON） | P1 |
| AC-9 | 模块化拆分（采集器、清洗器、报告生成分离） | P1 |
| AC-10 | 现有 cron job 无需修改即可运行 | P0 |
| AC-11 | JSON 输出结构向后兼容 | P0 |

## 非目标

- 不更换数据源（仍以 Exa API 为主力）
- 不更换推送渠道（仍使用飞书）
- 不实现实时推送（仍为每日定时）

## 风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Exa API 查询增加导致超时 | 采集失败 | 并行查询 + 超时控制 |
| 模块化拆分引入 bug | 功能异常 | 保留 v5 快照 + 逐模块测试 |
| 分类规则冲突 | 分类错误 | 优先级机制 + 测试用例 |

## 技术选型

| 组件 | 当前 | 目标 | 理由 |
|------|------|------|------|
| 代码结构 | 单文件 | 多模块 | 可维护性 |
| 配置 | 硬编码 | YAML | 灵活性 |
| 日志 | print | logging | 可观测性 |
| 类型 | 无 | 类型注解 | 代码质量 |
| 测试 | 无 | pytest | 质量保障 |
